# ANÁLISIS COMPLETO DEL PROYECTO EVA 2.0 - ENTORNO VIRTUAL DE APRENDIZAJE

## INFORMACIÓN GENERAL DEL PROYECTO

**Nombre del Proyecto:** EduSearch - Entorno Virtual de Aprendizaje
**Versión:** 2.0
**Tipo:** Plataforma educativa completa
**Arquitectura:** Full-stack (Frontend + Backend)

### Descripción General
Sistema completo de gestión de aprendizaje en línea con paneles diferenciados para administradores, profesores y estudiantes. Incluye gestión de cursos, evaluación de conocimientos, sistema de chat, notificaciones y recursos multimedia.

---

## ARQUITECTURA TÉCNICA

### Frontend (Next.js)
- **Framework:** Next.js 16 con App Router
- **Lenguaje:** TypeScript
- **UI Framework:** React 19 con shadcn/ui y Radix UI
- **Estilos:** Tailwind CSS v4
- **Gestión de Estado:** React Context API + React Query (TanStack)
- **Internacionalización:** react-i18next
- **Puerto:** 3000 (desarrollo), configurable en producción

### Backend (Spring Boot)
- **Framework:** Spring Boot 3.5.8
- **Lenguaje:** Java 21
- **ORM:** JPA/Hibernate
- **Base de Datos:** PostgreSQL
- **Seguridad:** Spring Security + JWT
- **Puerto:** 8080
- **Arquitectura:** MVC con capas de Controller, Service, Repository

### Infraestructura Adicional
- **Base de Datos:** PostgreSQL (puerto 5432)
- **Almacenamiento:** MinIO (puerto 9000/9001)
- **Monitoreo:** Prometheus (puerto 9090) + Grafana (puerto 3001)
- **Emails:** MailHog (puerto 1025/8025)
- **Contenedorización:** Docker + Docker Compose

## FUNCIONALIDADES PRINCIPALES

### 1. AUTENTICACIÓN Y REGISTRO
**Ubicación:** `app/auth/`, `components/auth/`, `lib/services/auth.service.ts`

#### Componentes que conforman la funcionalidad:
- `app/auth/login/page.tsx` - Página de inicio de sesión
- `app/auth/register/page.tsx` - Página de registro
- `components/auth/login-form.tsx` - Formulario de login
- `components/auth/register-form.tsx` - Formulario de registro
- `lib/context/auth.context.tsx` - Contexto global de autenticación
- `lib/services/auth.service.ts` - Servicio de API para autenticación
- `middleware.ts` - Middleware de Next.js para protección de rutas
- `components/layout/protected-route.tsx` - Componente de protección de rutas

#### Flujo de comunicación Frontend-Backend:
1. **Registro:**
   - Frontend: `POST /api/auth/register` con datos de usuario
   - Backend: Valida datos, encripta password, guarda usuario, genera JWT
   - Respuesta: Token JWT + datos de usuario

2. **Login:**
   - Frontend: `POST /api/auth/login` con email/password
   - Backend: Valida credenciales, genera JWT si correcto
   - Respuesta: Token JWT + datos de usuario

3. **Verificación de sesión:**
   - Frontend: `GET /api/auth/me` con token en header
   - Backend: Valida JWT y retorna datos del usuario actual

#### Seguridad implementada:
- JWT tokens almacenados en cookies HTTP-only
- Validación de roles en rutas protegidas (middleware Next.js)
- Encriptación BCrypt para contraseñas
- Protección CSRF
- Rate limiting en backend
- Validación de inputs en formularios
- Sanitización de datos

### Instalación Paso a Paso

#### 1. Clonar el Repositorio
```bash
git clone <url-del-repositorio>
cd eva2.0
```

#### 2. Configurar Variables de Entorno
Crear archivo `.env.local` en la raíz del proyecto:
```env
NEXT_PUBLIC_API_URL=http://localhost:8080/api
```

#### 3. Instalar Dependencias del Frontend
```bash
npm install
```

#### 4. Configurar PostgreSQL
```sql
-- Crear base de datos
CREATE DATABASE eva_db;

-- Crear usuario (opcional)
CREATE USER eva_user WITH PASSWORD 'eva_password';
GRANT ALL PRIVILEGES ON DATABASE eva_db TO eva_user;
```

#### 5. Configurar Backend
- Verificar `backend/src/main/resources/application.properties`
- Ajustar configuración de BD si es necesario

#### 6. Ejecutar con Docker (Método Recomendado)
```bash
# Desde la raíz del proyecto
docker-compose up -d
```

Esto iniciará:
- PostgreSQL en puerto 5432
- Backend Spring Boot en puerto 8080
- Frontend Next.js en puerto 3000
- MinIO en puertos 9000/9001
- Prometheus en puerto 9090
- Grafana en puerto 3001
- MailHog en puertos 1025/8025

#### 7. Ejecutar Manualmente (Alternativo)

**Backend:**
```bash
cd backend
mvn spring-boot:run
```

**Frontend:**
```bash
npm run dev
```

#### 8. Acceder a la Aplicación
- **Aplicación principal:** http://localhost:3000
- **Backend API:** http://localhost:8080/api
- **MinIO Console:** http://localhost:9001 (admin/admin)
- **Grafana:** http://localhost:3001 (admin/admin)

## MEDIDAS DE SEGURIDAD IMPLEMENTADAS

### Autenticación y Autorización
- **JWT Tokens:** Firmados y verificados en cada petición
- **Refresh Tokens:** Para mantener sesiones activas
- **BCrypt:** Encriptación de contraseñas con salt
- **Role-based Access Control:** Validación de permisos por endpoint
- **Session Management:** Stateless con JWT

### Protección de API
- **Rate Limiting:** Control de peticiones por IP/usuario
- **CORS:** Configurado solo para orígenes permitidos
- **Input Validation:** Validación de datos en backend y frontend
- **SQL Injection Prevention:** Prepared statements con JPA
- **XSS Protection:** Sanitización de inputs
- **CSRF Protection:** Tokens en formularios

### Seguridad de Datos
- **Encriptación en tránsito:** HTTPS obligatorio en producción
- **Almacenamiento seguro:** MinIO para archivos multimedia
- **Auditoría:** Logs de acciones sensibles
- **Data Sanitization:** Limpieza de datos antes de guardar

### Seguridad de Infraestructura
- **Container Security:** Imágenes Docker oficiales y actualizadas
- **Secrets Management:** Variables de entorno para credenciales
- **Network Security:** Comunicación interna entre contenedores
- **Monitoring:** Prometheus/Grafana para detectar anomalías

## CONCLUSIÓN

Este proyecto EVA 2.0 representa una plataforma educativa completa y robusta, diseñada con las mejores prácticas de desarrollo moderno. La arquitectura separa claramente responsabilidades entre frontend y backend, implementa medidas de seguridad exhaustivas, y está preparada para escalar según las necesidades de crecimiento.

La documentación completa y la configuración Docker facilitan el despliegue tanto en desarrollo como en producción. El sistema de roles permite una gestión granular de permisos, y las funcionalidades educativas cubren todos los aspectos necesarios para un entorno virtual de aprendizaje efectivo.

Para cualquier persona que desee trabajar con este proyecto, se recomienda seguir la guía de instalación paso a paso y familiarizarse primero con la estructura de módulos antes de realizar modificaciones.
